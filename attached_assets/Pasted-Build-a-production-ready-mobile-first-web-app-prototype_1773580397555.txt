==================================================
CORE IDEA
==================================================

Luck Miner is a bright, polished, addictive casual risk game.
The player taps tiles on a 4x4 mine board.
Tiles can reveal:
- coins
- relics
- multipliers
- traps

The player can cash out before hitting a trap.
The game should feel premium, shiny, fair, and intuitive.

==================================================
GAME RULES
==================================================

Board:
- 4 columns × 4 rows
- 16 tiles total

Tile types:
- coin
- relic
- multiplier
- trap

Coin tiles:
- give coins to current haul
- example values can be chosen randomly from a small set like:
  8, 10, 12, 15, 18, 20, 24, 30

Relic tiles:
- add relic count to permanent balance
- usually +1, sometimes +2

Multiplier tiles:
- increase current run multiplier
- use values like:
  x1.2, x1.4, x1.6, x1.8

Trap tiles:
- if no shield: run ends, haul is lost
- if shield exists: consume shield, protect player, continue run

==================================================
FREE PLAYS / DAILY RESET
==================================================

- player gets 8 free plays per day
- show free plays left clearly under the board
- after free plays are used, each extra play costs 40 coins
- show text like:
  “Free plays left today: X/8 · extra play costs 40 coins · resets in HH:MM:SS”

Reset behavior:
- reset happens at the same exact daily reset time regardless of whether plays were used
- for this prototype, implement daily reset using localStorage + timer
- structure code so this can later be moved to backend for Telegram production

When the player presses Start Dig / Play Again:
- if free plays remain, consume 1 free play
- otherwise, if coins >= 40, subtract 40 coins and start run
- otherwise do not start run

==================================================
RISK SYSTEM
==================================================

The game must feel fair, not brutal.

Use TWO risk layers:

1. Run Risk
- rises during a single run as the player keeps digging
- makes continued digging riskier

2. Streak Heat
- rises after successful cashouts across runs
- makes future runs slightly riskier overall
- prevents endless easy win streaks

IMPORTANT:
When a trap is hit, BOTH the run risk and the streak heat should drop.
This must happen whether:
- the trap ends the run
- OR a shield blocks the trap

So the game should NOT just get harder forever.

Use these approximate values:
- BASE_RISK = 4.5%
- POST_TRAP_RISK = 5.5%
- MAX_RISK = 26%
- RISK_STEP per safe dig = 1.3%
- CASHOUT_PRESSURE_STEP = 1%
- CASHOUT_PRESSURE_DROP_ON_TRAP = 2%
- MAX_CASHOUT_PRESSURE = 8%

Behavior:
- starting a run should begin near base risk + current streak heat
- every safe dig increases run risk a bit
- every successful cashout increases streak heat a bit
- every trap hit reduces streak heat and reduces the next baseline risk
- this should feel like:
  - winning a lot = the mine pushes back
  - hitting traps = pressure cools down

Show visible badges for:
- Steps
- Risk %
- Streak Heat %
- Shield count
- Best score
- Prototype badge
